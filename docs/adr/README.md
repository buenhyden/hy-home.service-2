# Architecture Decision Records (ADRs)

## ❓ What is an ADR?

An **Architecture Decision Record (ADR)** is a document that captures an important architectural decision made along with its context and consequences.

## 📂 Structure

ADRs are numbered sequentially and stored in this directory.

- `0001-record-architecture-decisions.md` (Example)
- `0002-use-nextjs-for-frontend.md`
- `0003-adopt-postgresql.md`

## 📝 How to Create an ADR

1. Copy the template (if available) or follow this format:
   - **Title**: Short and descriptive.
   - **Status**: Proposed, Accepted, Deprecated, etc.
   - **Context**: The issue motivating the decision.
   - **Decision**: The change that we are making.
   - **Consequences**: What becomes easier or more difficult.

2. Submit a Pull Request for team review.
