# Product Requirement Documents (PRDs)

## ❓ What is a PRD?

A **Product Requirement Document (PRD)** outlines the purpose, features, behavior, and success metrics of a new product feature associated with this project. It serves as the compilation of truth for "What" we are building.

## 📂 Organization

PRDs should be named clearly, often prefixed with the EPIC or Feature name.

- `auth-system-prd.md`
- `dashboard-v2-prd.md`

## 📝 How to Create a PRD

1. **Use the Template**: Copy `templates/prd-template.md` to this directory.
2. **Define Requirements**: Use the `[REQ-PRD-xxx]` ID format for traceable requirements.
3. **Review**: Ensure stakeholders approve the PRD before technical implementation begins.
