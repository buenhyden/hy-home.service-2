# Developer Guides

This directory contains practical, step-by-step guides for common development tasks.
These guides complement the **Workflows** found in `.agent/workflows/`. While workflows are instructions for AI agents, these guides are written for human developers to understand the "Why" and "How".

## 📂 Categories

### [Global](./Global/) (Planned)

General development tasks, git workflows, and troubleshooting.
*Ref: `.agent/workflows/Global`*

### [DevOps](./DevOps/) (Planned)

Deployment, CI/CD, and infrastructure management.
*Ref: `.agent/workflows/DevOps`*

### [Frontend](./Frontend/) (Planned)

React, Next.js, and web-specific guides.
*Ref: `.agent/workflows/React`, `.agent/workflows/Nextjs`*

### [Backend](./Backend/) (Planned)

API development, Database management, and Security.
*Ref: `.agent/workflows/Backend`, `.agent/workflows/Database`*

---

## 📝 Creating a New Guide

1. Identify the matching **Workflow** in `.agent/workflows/`.
2. Use the standard naming convention: `[category]-[action]-[topic].md`.
   - Example: `setup-sentry-monitoring.md`
3. Follow the standard template defined in `../documentation-standards.md`.
